# 关于Rust深度学习框架的想法

> [!NOTE]
> 想试试这个想法的人，尽管去做吧！
> 我唯一的要求是在你的README.md里附上这个仓库的链接。

现在，Rust已经有了很多的机器学习框架。但是，大多数的框架基本可以视作是对pytorch的重实现。
尽管pytorch确实是一个很优秀的框架，很灵活，性能也很不错，但依然受到动态图的制约。
而Tensorflow等框架使用静态图，但是并不方便。

## 自动生成网络
B站上已经有UP主进行了一些尝试，他写了一个蓝图引擎，让你可以像搭积木一样搭建模型，并自动生成代码，用于训练和推理。
这确实是一个很好的主意，但是当模型变大，节点和连线变多，就会有点力不从心。
当你面对一堆乱七八糟的线和节点时，心情都不好了。
并且，这样的引擎，扩展起来也不方便，给它写插件可能会遇到各种问题，包括渲染。
这是一个糟点。作为一个提升效率的工具，你却要面对这种开发上的噪声。
并且，它所用的项目文件不太能让人类阅读，所以为了读这个网络，我必须安装这个软件。
所以很自然的，我们希望有一种基于效率同样高、可读性一样好、尽可能直观，且人类可读的框架。

## Rust过程宏
Rust的过程宏是很强大的功能，它让你能在编译期生成代码，且syn quote proc-macro2这样的轮椅crate
让写过程宏变得比较舒服，而且还有一些更逆天的crate，做得更加轮椅。
同时，过程宏也很易于阅读。

## 介绍：基于Rust过程宏的深度学习框架
过程宏使轻松创造一个上述的框架变得更加简单。使用过程宏，你可以直接收到来自rust工具链的好处。
你不需要给你的框架写一个专门的插件系统，而是直接与cargo集成。插件开发者也不必面对渲染问题。
同时，你也不必面对一堆乱糟糟的线和节点，而是直接从最上层开始，借助rust-analyzer的功能在代码中跳转。
你甚至可以随便把模型的定义代码拆到不同的文件里，完全不受影响。
同时，文档的编写和阅读也更加方便，可以直接集成在模型的定义里。

## 给实现者看的一些提示
因为过程宏并不能获得源代码的全局信息，这对于静态图的优化不利。
所以我们可以把定义模型的代码塞到build.rs里，由过程宏基于这些代码生成一个生成器，
由这个生成器来在build.rs的运行时生成原始计算图，然后进行优化，再储存到一个文件里。
然后在main.rs里，可以用另一个过程宏读取这个文件里的计算图并生成对应的代码。

## 例子
帮你理解一下。

### MLP
```rust
#[model(in_dim = 10, out_dim = 1)]
struct Mlp {
    #[layer(in_dim = 10, out_dim = 20)]
    layer1: Linear,
    #[layer(foreach)]
    f1: relu,
    #[layer(in_dim = 20, out_dim = 10)]
    layer2: Linear,
    #[layer(foreach)]
    f2: relu,
    #[layer(in_dim = 10, out_dim = 1)]
    layer3: Linear,
    #[layer(foreach)]
    f3: sigmoid,
}
```
